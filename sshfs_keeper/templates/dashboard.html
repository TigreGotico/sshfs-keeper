A transfer is a one-shot copy or move between two paths — local folders, SSH servers, or rclone remotes. Unlike Sync jobs it runs once immediately and is not repeated on a schedule. For SSH hosts, rsync (SSH) is the best choice: it transfers only changed bytes, verifies checksums, and can resume interrupted transfers.

Get notified when a mount goes down or comes back up. Paste any HTTP webhook URL — works with ntfy.sh (free push notifications to your phone), Slack, Discord, or any service that accepts a JSON POST.

ntfy.sh is the easiest option: go to ntfy.sh, pick a topic name, paste the URL here, and install the ntfy app on your phone. For Slack: create an Incoming Webhook in your workspace settings. For Discord: Server Settings → Integrations → Webhooks.

JSON payload sent to the webhook: {"event":"failure"|"recovery"|"backoff","mount":"name","error":"...","timestamp":"..."}

These settings control how aggressively sshfs-keeper watches your mounts and retries failed connections. The defaults work well for most home networks.

seconds How often to test whether each mount is alive. Lower = faster detection but more overhead. 30s is a good default.
seconds How long to wait before attempting to remount after a failure. Gives the server time to come back up.
attempts After this many consecutive failures, sshfs-keeper enters backoff mode and retries less frequently to avoid hammering an offline server.
seconds Starting wait time in backoff mode. Doubles after each failed attempt (60s → 120s → 240s…). The mount will still remount automatically once the server returns.
INFO is recommended. Use DEBUG to see every check and connection attempt (very verbose). Use WARNING or ERROR to only log problems.

Add Mount

Make a remote folder appear as a local drive on this machine. sshfs-keeper will keep it mounted and automatically reconnect if the connection drops.

A short label you'll recognise — like nas, backup-drive, or work-server. No spaces.
Select a saved host, or choose "manual" to enter a raw connection string.
The remote folder you want to mount.
Format: username@server-address:/folder/path
This is the folder you'll open on this machine to access the remote files. It will be created automatically if it doesn't exist. Example: /mnt/nas or /home/you/nas
An SSH key lets you connect without typing a password. If you haven't set one up yet, leave this blank — it will use your system's default key. Upload keys in the SSH Keys tab.
sshfs is the default for SSH connections. rclone is required for FTP, WebDAV, SMB, and cloud storage. The Protocol selector above sets this automatically.
Comma-separated sshfs/FUSE options. The defaults enable caching, compression, and automatic reconnection — most users should leave this as-is. Ignored when using rclone.

Add Sync Job

Automatically copy files from one place to another on a schedule. Great for backups, mirroring, or keeping two folders in sync.

A label you'll recognise — like photos-backup or docs-mirror.
Select the host where the source files are, or "local" for folders on this machine.
Select where to copy the files, or "local" for a folder on this machine.
minutes sshfs-keeper will run the sync roughly every N minutes. Set to 1440 for once a day.
rsync is the right choice for most situations. lsyncd reacts to changes in real time instead of waiting for the next scheduled run. rclone is needed for cloud storage.
-az = archive mode + compression. --delete removes files from the destination that no longer exist at the source (mirror mode). Remove --delete if you want to keep old files.
Only needed if source or destination is a remote server and you want to use a specific key.
